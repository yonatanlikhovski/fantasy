# Quality Plots

High-quality, presentation-ready figures for the **Fantasy Value Prediction for
NBA Players** project, generated by
[`../generate_visualizations.py`](../generate_visualizations.py) from the CSV
outputs under [`../../sim_stats/`](../../sim_stats).

### Regenerating

Run from the repo root:

```bash
py visual_helper/generate_visualizations.py            # all themes
py visual_helper/generate_visualizations.py --theme value   # one theme only
```

Figures are saved as 200 dpi PNGs, grouped into five themes (one folder each).
All figures follow the course's visualization principles (Scale, Conditioning,
Perception, Transformation, Context, Smoothing): take-away titles, labelled axes,
reference lines, annotated outliers, captions, a colorblind-safe palette
(blue/orange, no red+green) and perceptually-uniform colormaps (viridis).

`Visualization_Insights.docx` (in this folder) explains, in English, what each
figure shows and the concrete take-away to cite in the writeup. Regenerate it with:

```bash
py visual_helper/build_insights_doc.py
```

## `overview/` - data sanity checks

These extend the two visualizations from milestone 2 and confirm the scraped
data is complete and reasonable. Source: `sim_stats/player_season_stats.csv`.

- **`active_players_per_season.png`** - Bar chart of the number of distinct
  players (`player_id`) active in each season (2023-2026). Acts as a sanity check
  on scraping completeness: the count should drift down by roughly 50 records per
  earlier season as newer players had not yet entered the league.

- **`player_retention.png`** - For every season, how many of the players active
  in the latest season (2026) were *also* active that season (set overlap on
  `player_id`), shown as counts and percentages. Verifies the expected ~70
  players-in / ~70 players-out turnover per year.

- **`ppg_distribution_by_season.png`** - A 2x2 grid of points-per-game
  (`pts_mean`) histograms, one per season, each with an overlaid normal-distribution
  fit (scipy) and a mean line. Shows the stretched-Gaussian shape described in the
  milestone: a peak around 6 points, a long right tail, and few players above 25.

## `value/` - fantasy H2H value insights

How the H2H fantasy value is built from the nine scoring categories. Sources:
`sim_stats/h2h_value_2027.csv` and `sim_stats/player_season_stats.csv`.

- **`category_breakdown_top15.png`** - Heatmap of the top 15 players by `rank_H2H`
  against their normalized score (0-1) in each of the nine categories
  (`PTS, REB, AST, STL, BLK, FG3M, TOV, FG, FT`), so you can see *where* each elite
  player's fantasy value comes from. A heatmap is used instead of a stacked bar to
  avoid the hard-to-read "jiggling baseline" of stacked segments. Turnovers are
  stored as negative because fewer is better.

- **`category_scarcity.png`** - Bar chart of each category's coefficient of
  variation (std / mean) across all players. Higher bars mean the stat is scarcer
  and more spread out (e.g. blocks, steals), which is why those categories swing a
  weekly head-to-head matchup more than common stats like points.

- **`category_correlation_heatmap.png`** - Pearson correlation matrix between the
  nine category scores across players. Reveals which categories tend to come
  together (e.g. blocks and rebounds) and which are independent, useful when
  building a team around specific strengths.

- **`top30_h2h_value.png`** - Horizontal bar ranking of the 30 highest `H2H_value`
  players, the core output of the valuation model.

## `thesis/` - real value vs fantasy value

The central question of the project: a player's fantasy value does not always
match his real-life value. Source: `sim_stats/ws_vs_h2h_2027.csv`.

- **`ws_vs_h2h_scatter.png`** - Scatter of Win Shares (real-life basketball value,
  x-axis) against H2H value (fantasy value, y-axis), with a regression line and the
  Pearson `r`. The most divergent players (largest residuals) are labelled. Points
  far from the line are valued very differently in fantasy than in reality.

- **`over_under_valued.png`** - Diverging horizontal bar of the players with the
  largest `rank_diff` (Win-Shares rank minus H2H rank). Green bars are overvalued
  in fantasy relative to real life; red bars are undervalued, surfacing potential
  draft bargains and traps.

## `model/` - projection model evaluation

How well the 2026 projections matched what actually happened. Sources:
`sim_stats/evaluation_2026.csv` and `sim_stats/evaluation_2026_summary.csv`.

- **`predicted_vs_actual.png`** - A 3x3 grid of predicted-vs-actual scatter plots
  for the nine stats (PTS, REB, AST, STL, BLK, TOV, FG3M, FG%, FT%), each with a
  `y = x` perfect-prediction reference line and the per-stat correlation. Tight
  clustering around the diagonal indicates accurate projections.

- **`error_mae_rmse.png`** - Grouped bars of mean absolute error (MAE) and root
  mean squared error (RMSE) per stat, in each stat's own units. RMSE above MAE
  flags stats with occasional large misses.

- **`correlation_pearson_spearman.png`** - Grouped bars of the Pearson and Spearman
  correlations between projected and actual values per stat (0-1 scale). Volume
  stats (points, rebounds, assists) rank highest; percentage stats are hardest to
  predict.

- **`ci_calibration.png`** - Bar of the share of actual values that fell inside the
  predicted 5-95% confidence interval per stat, against the ideal 0.90 line. Shows
  whether the model's uncertainty estimates are well calibrated, too narrow, or too
  wide.

## `draft/` - draft simulation results

Outcome of the simulated H2H draft. Sources: `sim_stats/draft_standings_2027.csv`
and `sim_stats/draft_team_averages_2027.csv`.

- **`standings_wins.png`** - Horizontal bar of each team's average wins (`W`),
  annotated with category points (`cat_pts`). The final league standings.

- **`team_category_strengths.png`** - Heatmap of teams (rows, ordered best-to-worst
  by standings) against the nine categories (columns). Values are per-category
  z-scores, so each cell shows how strong a team is in that category relative to the
  league. Turnovers are sign-flipped so that warmer/higher always means better,
  making each team's strategic strengths and weaknesses easy to read.
